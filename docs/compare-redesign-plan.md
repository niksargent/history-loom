# Compare Redesign Plan

## Purpose

Compare should explain **why two periods belong in the same conversation**.

It should not lead with overlap taxonomy.
It should lead with the echo story.

## Core rule

When a curated echo exists:
- lead with the authored echo claim
- explain what returns
- explain what changes this time
- use shared forces as the main evidence layer

When no curated echo exists:
- give a lighter comparison read
- stay honest if the pairing is more contrast than rhyme

## Chosen structure

Compare now follows this order:
1. summary cards
2. echo-story hero with bridge visual
3. population band
4. shared pressure mix
5. supporting evidence sections
6. event grounding

## Visual decision

The top compare visual is a **bridge**, not a Venn diagram.

It should show:
- left period
- right period
- the returning pattern in the centre
- stronger forces on each side

This makes the connection feel guided and historical, not analytical for its own sake.

## Language rule

Compare should not use public phrases such as:
- `curated echo link`
- `motif`
- `emergence`
- `fraying`
- `force in view`

Public copy should say:
- what returns
- what changes
- why it matters

## Evidence rule

Shared pressure is now the main evidence layer.

That means:
- it should sit above lower bucket sections
- it should be visual, not just listed
- it should make strong echoes feel legible even when mood or value overlap is lighter

## Status

This redesign is now live in the compare overlay.

The next compare task is not another structural rewrite.
It is live review:
- check whether the hero copy teaches clearly
- check whether the bridge reads fast enough
- check whether the shared pressure mix carries enough explanatory weight

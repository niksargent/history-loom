# USA Dataset Charter

## Purpose

Define dataset 2 for The History Loom as the United States, so the product can move beyond Britain while staying accessible to a broad public audience.

## Proposed dataset

- dataset name: `United States, 1776-2025`
- short label: `USA`
- scope: `United States`
- intended status for first release: `seed`, moving to `usable`

## Why this dataset

The United States is a strong second dataset because it combines:
- high public familiarity
- strong recurring tensions around legitimacy, expansion, inequality, race, citizenship, media, and polarization
- a large number of structurally interesting echoes within its own history
- obvious future comparison value against Britain and, later, France

It is a better next step than a more niche dataset because it helps the product prove:
- multi-dataset reuse
- broad audience appeal
- stronger answers to `does history repeat itself?`

## Audience value

For a non-specialist audience, the USA dataset should be able to illuminate:
- founding and constitutional order
- slavery and racial hierarchy
- civil war and reconstruction
- industrial capitalism and reform
- mass democracy and mass media
- postwar order, fragmentation, and polarization

## Core historical questions

The dataset should help users explore:
- when legitimacy expands and when it fractures
- how liberty and exclusion coexist
- how media and information systems reshape political life
- when reform stabilizes the system and when it fails
- how expansion, war, and internal conflict remake the state

## Planning assumption for the default lens

The working assumption is:
- one default civic lens
- comparable period blocks
- exact period boundaries to be finalized during source-backed seeding

This is intentionally still open.
It is a high-reasoning decision because the current tool favors a strong lens, but the USA dataset should not be distorted merely to mimic Britain.

## First-release maturity target

The first USA release should aim for:
- `seed` quickly
- `usable` before any cross-dataset compare is exposed

It does not need to begin at `rich`.

## Scope boundaries

The first USA version should:
- stay national in scope, with global context only where necessary
- reuse the current pressure model unless a clear gap emerges
- avoid overpromising on cross-dataset comparison before the data is mature enough

## Immediate next action

Produce the source-backed USA seeding plan and then create the first seed layer:
- periodization proposal
- first event candidates
- first pressure grounding candidates
- first snapshot approach

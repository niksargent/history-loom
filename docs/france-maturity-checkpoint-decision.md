# France Maturity Checkpoint Decision

## Decision

The France dataset is accepted as:

`template dataset 3 at usable-draft maturity`

It should not stay in guarded-draft limbo.
It should move into:
- selective validation
- selective correction where a real weakness appears in use
- reuse as the third working dataset in the product

## Why this decision is justified

France now has:
- a stable dataset charter and lens choice
- working app-path integration
- a source-backed runtime draft seed
- a completed validation review
- a completed period-quality review
- snapshots and lived voice across all periods
- a set of strong anchor eras for public use

It does **not** yet have:
- fully mature echo confidence across every period
- a totally even level of legibility across the full field
- cross-dataset-compare readiness
- a fully validated canonical status

That is acceptable.
The checkpoint is not saying France is finished.
It is saying France is mature enough to stop being treated as a special-case draft field.

## What this means

### France should now be treated as

- a live, usable draft dataset
- stable enough for product use
- still subject to selective validation

### France should not now be treated as

- a field that needs broad expansion by default
- a cross-dataset compare baseline
- a fully trusted canonical dataset

## Remaining France watchpoints

The most important watchpoints remain:
- `fr-p03`
- `fr-p06`
- `fr-p07`
- the more comparative/exploratory echoes

These are now maintenance and validation issues, not reasons to hold the dataset below usable-draft maturity.

## Immediate consequence

The product now has:
- Britain as template dataset 1
- USA as template dataset 2 at usable-draft maturity
- France as template dataset 3 at usable-draft maturity

That is enough to support the next multi-dataset phase without pretending that cross-dataset compare is ready.

## Next recommendation

Do not reopen France enrichment by default.

Proceed next with one of these:
1. selective validation across Britain, USA, and France
2. cross-dataset product planning without enabling comparison yet
3. selective strengthening only if live use reveals a real weak spot in `fr-p03`, `fr-p06`, or `fr-p07`

# Insights Lab Brief

## Purpose

`Insights Lab` is the separate reading surface for derived historical signals.

It exists so the main Loom page can stay sparse and visual while still giving users access to stronger analytical hooks.

The page is not:
- a research dashboard
- a statistics console
- a truth engine

It is:
- a curated discovery surface
- editorially framed
- confidence-banded
- linked back into the Loom for inspection

## Product role

The main page should surface one small hook.

Insights Lab should answer:
- what structural family a period belongs to
- which pressures travel together
- which periods look unusual
- later, which cross-dataset cousins deserve inspection

## Current v1 sections

### 1. Period families

Job:
- show recurring structural shapes inside one dataset

Desired read:
- `this period belongs to a broader family, not just itself`

### 2. Force relationships

Job:
- show strong co-movement, inverse, and one-step lead-lag signals

Desired read:
- `these pressures often travel together`

### 3. Outliers

Job:
- show periods that break the field’s usual pattern

Desired read:
- `this era is not just intense; it is structurally unusual`

### 4. Cross-dataset cousins

Job:
- later, show candidate structural cousins across countries

Current status:
- internal only
- not yet public in the Lab

## Interaction rules

The page should always let the user:
- go back to the Loom
- inspect a period in context
- inspect a force in context

The Lab should not trap the user inside abstract analysis.

## Tone rules

Keep the copy:
- short
- declarative
- non-causal
- editorial rather than technical

Avoid:
- statistical jargon in the main surface
- long method explanations in the content flow
- claims stronger than the data supports

Method detail belongs in docs, not in the main reader journey.

## Visual direction

The page should feel like a continuation of the existing exhibit surface:
- calm
- atmospheric
- restrained
- not a dashboard grid

This means:
- strong headings
- short claims
- confidence as quiet metadata
- one or two actions per card

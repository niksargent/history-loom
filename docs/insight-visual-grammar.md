# Insight Visual Grammar

## Purpose

Define how insight states should look and feel so the product communicates meaning visually before the user reads the text.

The key rule:
- insight should be seen as well as read

## Public-facing categories

### 1. Eras that behave alike

Meaning:
- periods that belong to the same recurring pattern

Visual treatment:
- warm, steady field tint
- grouped cards
- shared signal pills
- calm, banded or clustered feel

Public language:
- `Eras that behave alike`
- `This era belongs to a recurring pattern`

### 2. Forces that rise together

Meaning:
- two forces repeatedly move together, apart, or in sequence

Visual treatment:
- paired strands or rails
- cyan/amber emphasis depending on polarity mix
- closeness for together
- separation for apart
- staggered markers for leads

Public language:
- `Forces that rise together`
- `Forces that pull apart`
- `One force often moves before the other`

### 3. Moments that break the pattern

Meaning:
- outlier periods

Visual treatment:
- offset card treatment
- stronger interruption colour
- isolated marker or disturbed frame
- more visual separation from grouped cards

Public language:
- `Moments that break the pattern`
- `This era behaves differently from the ones around it`

### 4. Structural cousins

Meaning:
- cross-dataset periods with similar pressure signatures

Visual treatment:
- paired cards
- bridge line
- mirrored tones from both datasets

Public language:
- `Distant cousins`
- `These two eras feel alike in surprising ways`

## Strength cues

Strength should be visible through:
- line weight
- opacity
- glow intensity
- rail thickness
- confidence chips only as secondary support

Do not rely on the user reading `strong`, `moderate`, or `experimental` first.

## Relationship visuals

### Together
- parallel lines or braids
- shared glow
- minimal distance

### Apart
- diverging lines
- split centre
- visible widening

### Leads
- staggered pulses
- one line slightly ahead
- offset markers

## Whole-tool rule

This grammar should not live only in Insights Lab.
It should influence:
- Force Explorer
- Compare
- insight prompts
- future motion views

## Initial implementation order

1. Insights Lab
2. Detail-panel insight prompt
3. Compare insight strip
4. Force Explorer hints
5. later motion and cousin views

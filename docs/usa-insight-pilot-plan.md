# USA Insight Pilot Plan

## Purpose

Use the United States dataset as the first public pilot for the derived insight layer.

This is the right pilot because USA is:
- already live
- structurally rich
- methodologically cleaner than Britain
- less provisional than France for public derived surfacing

## Pilot scope

Publicly surface:
- period family assignments
- pressure relationships
- outlier periods
- one short prompt per selected period where the signal is strong enough

Do not publicly surface yet:
- cross-dataset cousins
- automated echo support claims
- weaker experimental relationships

Rollout scope:
- `single-dataset pilot`

Pilot end state:
- this pilot must either promote into `all live datasets` for the approved public insight surfaces
- or be explicitly limited and held back with a recorded reason

## Why USA first

USA already supports:
- clear regime and legitimacy shifts
- strong technology and information patterns
- useful inequality and institutional readings
- a manageable number of periods for legible clustering

It is a good dataset for proving that derived analysis can create stronger `aha` moments without making the product feel academic.

## Pilot outputs

### Main page

Add:
- one restrained derived prompt in the selected-period flow
- one route into Insights Lab

Do not add:
- multiple new analytical cards
- dense method explanation
- extra top-level clutter

### Insights Lab

Public USA sections:
- period families
- force relationships
- outliers

### Internal review

Generate but hold:
- cross-dataset affinities
- echo support scoring

## Editorial rules

USA surfaced prompts must:
- be short
- be structurally legible
- avoid causal language
- feel worth clicking

Bad prompt:
- `The model predicts growing instability`

Better prompt:
- `This era sits at the edge of the field’s usual pattern`

## Success criteria

The pilot is successful if:
- at least a few prompts feel genuinely revealing
- the main page gains hooks without feeling denser
- the Lab feels like a curated extension, not a dashboard
- the architecture is ready for Britain, France, and later cross-dataset release

## Promotion path

### Phase 1: keep USA public, tighten quality

Required now:
- public prompts use plain public language, not internal cluster labels
- the public Lab feels discoverable rather than technical
- the strongest USA prompts remain legible and worth clicking

### Phase 2: widen the same public surfaces to all live datasets

Promotion target:
- Britain and France receive the same public surface types as USA:
  - family groups
  - relationship groups
  - outlier groups
  - one restrained period prompt

Promotion gate:
- wording is plain enough for public use
- generated sections are visually and editorially coherent
- no dataset feels materially weaker just because the feature was piloted elsewhere first

### Phase 3: consider first public cross-dataset reveal

Only after Phase 2:
- review internal cousin candidates
- select one public cross-dataset pairing only if it is strong, legible, and genuinely revealing

Current best candidate:
- `USA 2001–2025` with `France 1981–2025`

### Failure / hold conditions

Hold or retire expansion if:
- prompts remain too repetitive
- family labels are not intelligible in public wording
- a dataset needs too much exception handling to fit the shared public surface

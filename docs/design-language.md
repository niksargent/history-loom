# Design Language — The History Loom

## 1. Design goal

The interface should feel premium, meditative, and intelligent. It should create a sense of calm structure hiding deep complexity.

## 2. Core principle

**Calm at rest. Deep in motion.**

The default state should be sparse and legible. Detail should appear only when invited.

## 3. Desired qualities
- elegant
- restrained
- cinematic but not flashy
- precise
- emotionally resonant
- quietly surprising

## 4. Surface behaviour

At rest, the interface should show only:
- core structure
- a few labels
- the selected lens
- the active time span

On interaction, it can reveal:
- detail cards
- event clusters
- gains and losses
- mood and value labels
- pressure traces
- echo threads

## 5. Visual direction

### Recommended palette direction
- dark charcoal / mineral base
- muted warm metallic accents
- soft off-white text
- selective use of richer colour only for active pressures or selected states

### Character
The interface should feel like a hybrid of:
- archive
- observatory
- woven instrument
- geological reading surface

## 6. Typography

### Recommended approach
- refined serif or humanist display for key headings
- clean sans-serif for UI labels, controls, and annotations

### Guidance
- labels should be short
- avoid long paragraphs in main view
- use sentence fragments where helpful
- let hierarchy do the work

## 7. Motion

Motion should feel like revealing hidden structure, not decorating the screen.

### Suitable motion types
- gentle unfolding
- slow tracing of a pressure line
- soft highlighting of echo threads
- controlled expansion of a selected period

### Avoid
- flashy transitions
- excessive bounce
- busy constant animation
- novelty motion that weakens seriousness

## 8. Information density rule

Never let the interface feel like a textbook exploded onto glass.

Use progressive disclosure everywhere.

## 9. Comparative design rule

Users should always be able to understand:
- where they are in time
- what lens is active
- why two eras are being compared
- what the claimed similarity or pressure relationship is

## 10. Emotional design rule

The interface must not feel sterile.
It should allow users to feel the tension between:
- gain and loss
- order and freedom
- confidence and overreach
- material progress and human cost
